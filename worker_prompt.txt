## What to change
The test file currently patches `requests.get` with `unittest.mock.patch`. After migration, the source modules use `httpx.AsyncClient` instead, so the mocks must change.

Current test pattern:
```python
from unittest.mock import patch, MagicMock

def test_get_user():
    with patch('src.api.users.requests.get') as mock_get:
        mock_response = MagicMock()
        mock_response.json.return_value = {"id": 1, "name": "Alice"}
        mock_response.raise_for_status.return_value = None
        mock_get.return_value = mock_response
        from src.api.users import get_user
        result = get_user(1)
        assert result["id"] == 1
```

New test pattern (migrate to):
```python
from unittest.mock import patch, AsyncMock, MagicMock
import asyncio

def test_get_user():
    async def run():
        mock_response = MagicMock()
        mock_response.json.return_value = {"id": 1, "name": "Alice"}
        mock_response.raise_for_status.return_value = None

        mock_client_instance = MagicMock()
        mock_client_instance.get = AsyncMock(return_value=mock_response)

        with patch('src.api.users.httpx.AsyncClient') as mock_client:
            mock_client.return_value.__aenter__.return_value = mock_client_instance
            from src.api.users import get_user
            result = await get_user(1)
            assert result["id"] == 1

    asyncio.run(run())
```

Apply the same pattern to `test_get_product()`:

- Use `AsyncMock` for async methods
- Wrap in `asyncio.run()` since test functions are sync
